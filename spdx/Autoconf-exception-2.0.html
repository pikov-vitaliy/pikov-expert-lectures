ПЕРЕВОД НА РУССКИЙ ЯЗЫК — Исключение Autoconf 2.0

Неофициальный русский перевод. Официальным и обязательным остается оригинальный текст лицензии.

В качестве особого исключения Фонд свободного программного обеспечения даёт неограниченное разрешение копировать, распространять и изменять скрипты configure, которые являются выводом Autoconf. Вам не нужно следовать условиям Стандартной общественной лицензии GNU при использовании или распространении таких скриптов, даже несмотря на то, что части текста Autoconf появляются в них. Стандартная общественная лицензия GNU (GPL) всё же регулирует всё иное использование материала, составляющего программу Autoconf.

Определённые части исходного текста Autoconf предназначены для копирования (в определённых случаях, в зависимости от ввода) в вывод Autoconf. Мы называем их частями «данных» (data). Остальной исходный текст Autoconf состоит из комментариев плюс исполняемого кода, который решает, какие из частей данных выводить в каждом конкретном случае. Мы называем эти комментарии и исполняемый код частями «не-данных» (non-data). Autoconf никогда не копирует какие-либо из частей не-данных в свой вывод.

Это особое исключение к GPL применяется к версиям Autoconf, выпущенным Фондом свободного программного обеспечения. Когда вы создаёте и распространяете изменённую версию Autoconf, вы можете распространить это особое исключение к GPL так, чтобы оно применялось и к вашей изменённой версии, *если только* ваша изменённая версия не имеет возможности скопировать в свой вывод некоторую часть текста, который был частью не-данных версии, с которой вы начали. (Иными словами, если только ваше изменение не перемещает или не копирует текст из частей не-данных в части данных.) Если ваша модификация имеет такую возможность, вы должны удалить любое уведомление об этом особом исключении к GPL из вашей изменённой версии.

Certain portions of the Autoconf source text are designed to be copied (in certain cases, depending on the input) into the output of Autoconf. We call these the "data" portions. The rest of the Autoconf source text consists of comments plus executable code that decides which of the data portions to output in any given case. We call these comments and executable code the "non-data" portions. Autoconf never copies any of the non-data portions into its output.

This special exception to the GPL applies to versions of Autoconf released by the Free Software Foundation.

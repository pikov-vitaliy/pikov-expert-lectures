# risk.pikov.expert — комплект для подготовки лендинга

Папка содержит исходники для подготовки авторского лендинга по **Дню I** (Риски ИБ, безопасность ЗО КИИ, инциденты) программы профессиональной переподготовки УЦ МАСКОМ. Лендинг разместится на поддомене **`https://risk.pikov.expert/`**.

## Состав

| Файл | Что это | Кому |
|------|---------|------|
| `Лекция Риски ИБ - материалы.md` | Большой markdown с полным содержанием 5-блочной лекции (5 × 1,5 ак. ч). Структурирован по расписанию дня, с глоссарием, перечнем нормативки и контрольными вопросами по каждому блоку. | Слушателям — как раздаточный материал; преподавателю — как исходник для презентации и для актуализации; Claude Designer — как источник содержания. |
| `Промпт для Claude Designer.md` | Готовый промпт-ТЗ для Claude (Code/Sonnet/Opus): создать `index.html` лендинга в формате слайдовой презентации (~100 слайдов) с режимом «лонгрид». Включает раскадровку всех слайдов, технические и дизайн-требования по образцу sast/astra-hardening. | Тому, кто будет генерировать `index.html` через Claude. |
| `README.md` | Этот файл. | — |

## Как использовать

1. **Получить `index.html`** через Claude:
   - откройте Claude.ai (или Claude Code в любой папке с записью);
   - вставьте в чат содержимое файла **`Промпт для Claude Designer.md`**;
   - в той же подаче приложите содержимое **`Лекция Риски ИБ - материалы.md`** туда, где в промпте стоит метка «(ВСТАВЬТЕ СОДЕРЖИМОЕ ФАЙЛА «Лекция Риски ИБ - материалы.md» НИЖЕ ПРИ ПЕРЕДАЧЕ ПРОМПТА В CLAUDE …)»;
   - дополнительно полезно приложить файлы-образцы стиля `sast/index.html` и `astra-hardening/index.html` из соседних папок;
   - Claude вернёт код `index.html`, готовый к открытию в браузере.

2. **Положить полученный `index.html`** в эту же папку `risk/`. Если нужны вспомогательные файлы (логотип, подложка) — добавить рядом.

3. **Раздать слушателям** файл `Лекция Риски ИБ - материалы.md` как полные раздаточные материалы (можно конвертировать в PDF через `pandoc Лекция... -o Лекция....pdf` или печатать прямо из браузера в лонгрид-режиме лендинга).

4. **Опубликовать лендинг** на поддомене `risk.pikov.expert` через тот же механизм, что для `sast.pikov.expert` и `astra-hardening.pikov.expert`.

## Что нужно проверить вручную перед публикацией

- **Актуальность нормативки.** Исходник подготовлен по состоянию на май 2026. Перед публикацией перепроверить, не вышли ли с тех пор новые редакции/приказы, особенно по:
  - 187-ФЗ (последние правки 58-ФЗ и 325-ФЗ);
  - приказ ФСТЭК 117 (с 01.03.2026);
  - методика критичности уязвимостей ФСТЭК (30.06.2025);
  - приказы ФСБ № 282, 213.
- **Соответствие фирменному стилю** (логотип, цвет, шрифты — Claude по умолчанию возьмёт за основу `sast/` и `astra-hardening/`).
- **Контакты:** `vitaliy@pikov.expert`, `@UnderLineSecurity`, ссылки на `pikov.expert` и `mascom-uc.ru` — корректны.
- **Регуляторы** упомянуты в кратких формах (ФСТЭК, ФСБ, Минцифры и т.п.), без слова «России» при повторных упоминаниях (это требование шаблона промптов автора).

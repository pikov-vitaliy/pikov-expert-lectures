# Open Source Readiness

Цель: превратить рабочую папку сайта в аккуратный публичный GitHub-репозиторий для сайта лекций, не публикуя временные артефакты, служебные данные хостинга и материалы с неясными правами.

## Рекомендуемое название

**`pikov-expert-lectures`**

Почему это хороший вариант:

- связывает репозиторий с доменом `pikov.expert`;
- сразу объясняет, что внутри лекции, а не монолитный сайт-комбайн;
- не запирает проект только в ИБ: можно добавлять ИС, ВКР, архитектуру ЭВМ и новые курсы.

## Рекомендованная подача

> Source repository for pikov.expert: open web lectures on information security, secure software development, SSDLC/RBPO, SCA/SBOM, operating systems, and applied computer science education.

По-русски:

> Открытый репозиторий сайта pikov.expert: веб-лекции по информационной безопасности, безопасной разработке, РБПО, SCA/SBOM, операционным системам и прикладным ИТ-дисциплинам.

## Что публиковать

Публиковать:

- корневой `index.html`, `.htaccess`, `robots.txt`, `sitemap.xml`;
- доменные папки с публичными HTML-страницами;
- авторские CSS/JS/изображения, если права понятны;
- `_PROJECT/lectures.json`;
- скрипты `_PROJECT/*.ps1` и `_PROJECT/*.mjs`;
- README, license/policy files, GitHub Actions.

Не публиковать:

- `_PROJECT/.browser-node`;
- `_PROJECT/.browser-qa-*`;
- `_PROJECT/.hosting-deploy-*`;
- `_PROJECT/.release-staging-*`;
- `release` и `**/release`;
- ZIP-архивы;
- `_НА_УДАЛЕНИЕ_*`;
- отчёты с локальными путями, сервером или рабочими выводами;
- материалы с неясными правами распространения.

## Предпубликационные gates

- `git status --short` показывает только ожидаемые файлы.
- `git check-ignore -v` подтверждает, что ZIP, release, staging, node_modules и карантин не попадут в историю.
- Локальная проверка проходит:
  `powershell -NoProfile -ExecutionPolicy Bypass -File .\_PROJECT\smoke-check.ps1`
- Секретов, приватных ключей и операционных идентификаторов хостинга нет:
  проверены приватные ключи, GitHub token prefixes, cloud access keys, API keys и SSH host/user identifiers.
- Отдельно решён вопрос с PDF/PPTX/DOCX и сторонними материалами.
- Финально выбрана лицензия:
  CC BY 4.0 для авторского учебного контента + MIT для скриптов, либо более строгая модель.

## Публикационная стратегия

### Вариант A. Полный публичный сайт после лицензионной чистки

Лучший баланс. В репозиторий идут исходники сайта, доменные папки, авторские материалы и проверенные сторонние материалы. Всё генерируемое исключается `.gitignore`.

### Вариант B. Безопасный public-core

Публикуются только HTML, CSS, JS, скрипты и ссылки на внешние первоисточники. PDF/PPTX/DOCX с неясными правами удаляются или заменяются ссылками. Это юридически самый спокойный вариант.

### Вариант C. Полное зеркало с Git LFS

Публикуется почти всё публичное содержимое сайта, включая бинарные материалы. Этот вариант требует Git LFS и явного подтверждения прав на каждый внешний документ. Не рекомендуется как первый шаг.

## Следующее решение

Рекомендуемый первый публичный релиз:

- repo name: `pikov-expert-lectures`;
- visibility: public;
- default branch: `main`;
- первая версия: `v2026.06`;
- license model: CC BY 4.0 for original content, MIT for scripts, third-party exceptions in `NOTICE.md`.

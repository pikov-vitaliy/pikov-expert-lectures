1 На server1 и infra проверить работоспособность службы ssh.
     При необходимости запустить службу ssh и добавить ее в автозагрузку.

      sudo systemctl status ssh
      sudo systemctl start ssh
      sudo systemctl enable ssh

2 На ВМ infra настроить SSH-подключение по ключам RSA к хосту server1
     под учетными записями sa. На ВМ infra выполнить следующие команды:
      ssh-keygen -t rsa
      ssh-copy-id sa@10.0.1.1

3 С ВМ infra зайти на ВМ server1 по ssh под пользователем sa, не вводя
     пароль. У вас должно получиться:
      ssh sa@10.0.1.1

4 На ВМ server1 настроить SSH-подключение по ключам RSA к хосту infra
     под учетными записями sa. На ВМ server1 выполнить следующие команды:
      ssh-keygen -t rsa
      ssh-copy-id sa@10.0.1.254

5 С ВМ server1 зайти на ВМ infra по ssh под пользователем sa, не вводя
     пароль. У вас должно получиться:
      ssh sa@10.0.1.254

6 Создать ВМ server2 путем клонирования ВМ template_1804_1_6.

7 В созданной ВМ server2 сменить имя хоста на server2.

      sudo hostnamectl set-hostname server2

8 Добавить в файл /etc/hosts запись для server2:
      127.0.0.1 localhost
      10.0.1.2 server2

9 Остановить и запретить запуск NetworkManager.

      sudo systemctl stop NetworkManager
      sudo systemctl mask NetworkManager

10 Настроить сетевой интерфейс eth0 в файле /etc/network/interfaces:

     • IP-адрес 10.0.1.2/24;

     • шлюз 10.0.1.254.

Переадресация портов

Задание: в первой части задания на хосте server1 запустить SOCKS-прокси
с помощью SSH-туннеля через хост infra и настроить работу утилиты apt
через SOCKS-прокси; во второй части задания на хосте server1 установить
и запустить веб-сервер Apache. Отключить на веб-сервере режим работы
AstraMode и обеспечить доступ к веб-серверу только по адресу 127.0.0.1.
На хосте infra с помощью SSH настроить переадресацию локального порта
для доступа к веб-серверу, работающему на хосте server1.

1 На ВМ server1 настроить динамическую переадресацию (SOCKS-прокси)
     на хост infra.

ssh -fN -D 1080 10.0.1.254

2 Проверить, что SOCKS-прокси запустился.

ss -tln | grep 1080

3 Настроить работу утилиты apt через SOCKS-прокси:
     • создать файл /etc/apt/apt.conf.d/70proxy;

     • добавить в файл настройки для прокси:
         Acquire::https::Proxy "socks5h://127.0.0.1:1080";
         Acquire::http::Proxy "socks5h://127.0.0.1:1080";

     • проверить, что apt работает через прокси.

sudo apt update

4 На ВМ server1 установить веб-сервер apache2.

sudo apt install apache2

5 Содержимое файла /var/www/html/index.html заменить следующим
     содержимым:

<HTML>

         <BODY>
         Course AL-1804
         </BODY>

</HTML>

6 Произвести следующие настройки apache2:

• в файле /etc/apache2/apache2.conf задать параметр

AstraMode off

• в файле /etc/apache2/ports.conf закомментировать параметр Listen 80
  и добавить

Listen 127.0.0.1:80
